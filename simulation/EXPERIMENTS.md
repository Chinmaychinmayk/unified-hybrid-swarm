# Experiments: Baseline quadrotor vs. Robotic fish

This document records a controlled comparison between the upstream quadrotor
swarm (**baseline**) and the robotic-fish variant (**fish**, see
[`ROBOTIC_FISH.md`](ROBOTIC_FISH.md)) on an identical formation-flight task
through a cluttered environment.

## What is being tested

The fish variant differs from the baseline only in its **dynamic-feasibility
envelope** — a slower cruise speed and a much gentler acceleration limit (the
appearance changes don't affect motion). The question is: *with the same
EGO-Swarm formation planner and the same obstacle field, how does shrinking the
dynamic envelope change the swarm's motion, timing, formation accuracy, and
safety?*

Everything except the envelope is held fixed:

| Held fixed | Value |
| --- | --- |
| Planner | EGO-Swarm formation planner (`ego_planner`), unchanged |
| Agents | 7, hexagon formation template |
| Start → goal | x = −26 m → +26 m (autonomous global-waypoint mode, `flight_type=2`) |
| Environment | `random_forest` obstacle field, **5 seeds** (1–5), identical per condition |
| Recording | 90 s per run at 100 Hz (`record_exp.py`), 6 s warm-up before recording |

| Varied (the only difference) | Baseline | Fish |
| --- | --- | --- |
| `max_vel` | 1.5 m/s | 1.0 m/s |
| `max_acc` | 8.0 m/s² | 2.5 m/s² |

## How to reproduce

Everything runs inside the `robotic-fish:latest` container. The host
`experiment/` dir is mounted at `/exp` and `experiment/results/` at `/out`.

```bash
# 1. data collection — sweeps both conditions × 5 seeds, writes /out/<cond>/seed<k>/
docker run -d --name fish_exp --network host \
  -v "$PWD/experiment":/exp -v "$PWD/experiment/results":/out robotic-fish:latest sleep infinity
docker exec -e REC_SECONDS=90 -e SEEDS="1 2 3 4 5" fish_exp bash /exp/run_all.sh

# 2. analysis — metrics, aggregate table, and figures
docker exec fish_exp python3 /exp/analyze.py
```

- `run_all.sh` edits `run_in_sim.launch` (vel/acc, `flight_type=2`) and
  `normal_hexagon.launch` (obstacle seed) per run, launches `roscore` +
  `normal_hexagon.launch`, records, then tears the ROS graph down.
- `analyze.py` reads `results/<cond>/seed<k>/{odom_i,cmd_i,obstacles}.csv`,
  computes per-run metrics, aggregates mean ± std across seeds, and writes the
  table + figures below. Mean speed/accel are taken over *moving* samples
  (|v| > 0.05 m/s) so the post-arrival stationary tail does not bias them.

**Recording window:** the window must outlast the slowest condition. The fish
need ≈ 64 s to traverse, so an earlier 60 s sweep left every fish run short of
the goal (traversal time undefined). The window was raised to 90 s, after which
both conditions complete with margin. The superseded 60 s run is preserved under
`experiment/results_60s_*/`.

## Results

Identical planner, identical environment (7 agents, hexagon, 5 obstacle seeds).
Only the dynamic-feasibility envelope differs.

| Metric | Unit | Baseline (quadrotor) | Robotic fish |
|---|---|---|---|
| Max speed | m/s | 1.581 ± 0.014 | 1.126 ± 0.047 |
| Mean speed | m/s | 0.976 ± 0.030 | 0.726 ± 0.068 |
| Max accel | m/s² | 1.355 ± 0.249 | 0.854 ± 0.383 |
| Mean accel | m/s² | 0.218 ± 0.025 | 0.123 ± 0.031 |
| RMS jerk | m/s³ | 0.658 ± 0.116 | 0.338 ± 0.129 |
| Path length | m | 51.420 ± 0.224 | 52.325 ± 0.372 |
| Traversal time | s | 44.919 ± 1.433 | 64.501 ± 4.327 |
| Formation RMSE (mean) | m | 0.176 ± 0.023 | 0.217 ± 0.026 |
| Formation RMSE (max) | m | 0.774 ± 0.078 | 0.799 ± 0.208 |
| Min inter-agent dist | m | 1.599 ± 0.354 | 1.705 ± 0.453 |
| Mean obstacle clearance | m | 1.116 ± 0.068 | 1.094 ± 0.077 |
| Near-miss fraction (< 0.3 m) | – | 0.058 ± 0.014 | 0.066 ± 0.013 |
| Success rate | % | 100 | 100 |

_n = 5 seeds per condition. ± is std across seeds. Source: `experiment/results/summary_table.{md,csv}`, per-run rows in `metrics_per_run.csv`._

## Findings

1. **Both swarms always succeed.** All 7 agents reach the goal in all 5 seeds
   for both conditions, with no inter-agent collisions (min separation ≈ 1.6–1.7 m).
2. **The fish move slower and far more gently.** Max/mean speed drop ≈ 29 %/26 %,
   and — the headline effect — **RMS jerk roughly halves** (0.66 → 0.34 m/s³)
   while mean acceleration drops ≈ 44 %. The tighter accel limit yields visibly
   smoother, lower-effort motion, consistent with drag-limited fish swimming.
3. **The cost is time, not quality.** The fish take ≈ 44 % longer to traverse
   (64.5 s vs 44.9 s) for an almost identical path length.
4. **Formation and safety are preserved.** Formation RMSE (mean and max),
   obstacle clearance, near-miss fraction, and inter-agent spacing are within
   noise of the baseline — the gentler envelope does not degrade how well the
   swarm holds shape or avoids obstacles; the planner simply spends more time.

## Caveats

- `n = 5` seeds — enough to separate the kinematic effects (which are large
  relative to their spread) but the near-equal formation/safety metrics are
  "within noise," not proven equal.
- This is the re-skin + fish-params level of fidelity: the underlying control is
  still the quadrotor (so3) model with reduced limits, not true non-holonomic
  swim dynamics (see the scope note in `ROBOTIC_FISH.md`). The kinematic
  differences reflect the envelope, not hydrodynamics.
- `fig_kinematics.png` shows only the first 20 s (the acceleration-rich phase);
  aggregate metrics in the table use the full run.
