# Robotic Fish Variant

This is a **robotic-fish re-skin** of the upstream `Swarm-Formation` quadrotor
swarm framework. The optimization/formation core is unchanged; what changed is
the *appearance* and the *dynamic-feasibility envelope* so the swarm behaves and
looks like a school of robotic fish instead of a swarm of drones.

## What was changed

| Area | File | Change |
| --- | --- | --- |
| Robot mesh | `src/Utils/odom_visualization/meshes/fish.stl` | New stylized fish mesh (laterally-compressed body, caudal + dorsal + pectoral fins), nose along **+x**, ~1.2 units long. Generated by `make_fish.py`. |
| Default skin | `src/Utils/odom_visualization/src/odom_visualization.cpp` | Default `mesh_resource` changed from `hummingbird.mesh` to `fish.stl`. |
| Per-robot skin + color | `src/planner/plan_manage/launch/simulator.xml` | `odom_visualization` node now loads `fish.stl` and is colored fish-orange (`rgb 0.95/0.55/0.10`). |
| Swim dynamics | `src/planner/plan_manage/launch/run_in_sim.launch` | `max_vel 1.5 → 1.0`, `max_acc 8.0 → 2.5` (slower cruise + gentle, drag-limited acceleration typical of a fish). |

> Scope note: this is the **re-skin + fish-params** level of conversion. The
> planner still uses the original quadrotor (so3) dynamics/control underneath.
> A higher-fidelity conversion (non-holonomic forward-swim + bounded turn rate,
> hydrodynamic drag/added-mass, buoyancy, BCF thrust, water currents) would
> additionally replace `so3_quadrotor_simulator` / `so3_control` and add a
> non-holonomic constraint term in `poly_traj_optimizer`.

## Regenerating the fish mesh

```bash
cd src/Utils/odom_visualization/meshes
python3 make_fish.py      # writes fish.stl (pure stdlib, no numpy needed)
```

## Build

Native ROS Noetic is only available on Ubuntu 20.04. On other systems use the
provided Docker image (the `Dockerfile` runs `catkin_make` for you):

```bash
make docker_build      # or: docker build -t robotic-fish:latest .
```

## Run (needs a GUI / X server for rviz)

```bash
xhost +
make docker_run        # starts the container with X forwarding
# inside the container:
source devel/setup.bash
roslaunch ego_planner rviz.launch          # window 1: visualization
roslaunch ego_planner normal_hexagon.launch  # window 2: the fish school
```

Then use **2D Nav Goal** in rviz to send the school a target (with
`flight_type = 3`), or set `flight_type = 2` to use the global waypoints baked
into `normal_hexagon.launch`. You will see seven orange robotic fish hold a
hexagonal school formation while weaving through the random-forest obstacles.
